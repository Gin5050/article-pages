今日の本命

Shepherd: A Runtime Substrate Empowering Meta-Agents with a Formalized Execution Trace

arXiv / Hugging Face Papers

これは、agent memory / skill / eval を個別部品として見るより一段下の「実行基盤」の論文として読める。 typed event による実行トレース、過去状態の fork / replay、supervisor による runtime intervention、counterfactual meta-optimization、Tree-RL rollouts という並びが、harness engineering の関心にかなり近い。

Direct Corpus Interaction が「検索インターフェースを太くする」話だとすると、Shepherd は「実行インターフェースを太くする」話に見える。 失敗した実行を眺めるだけでなく、巻き戻して別分岐を走らせる、介入する、学習データにする、という発想がある。

読む入口は、Abstract、system overview、3つの application。特に runtime intervention、counterfactual meta-optimization、Tree-RL training の結果だけ先に拾うと、論文の旨味に入りやすい。

Remember the Decision, Not the Description

memory を「過去の説明をどれだけ復元できるか」ではなく、「将来の意思決定に必要な区別をどれだけ保てるか」で見る。Storage Is Not Memory 系の続きとして強い。

NanoResearch

研究自動化を「誰にとっての自動化か」から入り、skill bank、memory、policy learning を個人の研究履歴に寄せる。

Dynamic Skill Lifecycle Management

skill をただ蓄積するのではなく、retain / retire / expand の lifecycle として扱う。skill rot の話に近い。

Rethinking Agentic Search with Pi-Serini

frontier LLM の agentic loop なら BM25 でも deep research に足りるのか、という実装寄りの問い。DCI の補助線になる。
