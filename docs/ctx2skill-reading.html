1. どんなもの？

Ctx2Skill は、長く複雑な文脈から、その文脈で何度も使える自然言語 skill を自動で作る手法です。 ここでいう文脈は、製品ドキュメント、臨床ガイドライン、実験データ、複雑な規則体系のように、 モデルが事前学習だけでは扱えない情報を指します。

重要なのは、文脈を単に要約するのではなく、次のタスクでも使える「手順」や「判断基準」として取り出すこと。 つまり、長い context を毎回読み捨てるのではなく、再利用可能な作業知に変換しようとしています。

HTML artifact の話に引き寄せるなら、これは「情報をどう見せるか」より奥にある、 「見た情報を次回の実行能力にどう変えるか」の論文です。

2. 先行研究と比べてどこがすごい？

一番の差分は、外部フィードバックがない状況で skill を作ろうとしているところです。 既存の skill 構築は、テスト結果、正解、タスク完了報酬のような外部信号に頼れることが多い。 でも文脈学習では、「この文脈から作った skill が十分か」を教えてくれる正解がありません。

Ctx2Skill はこの問題を、自己対戦の形に変えます。 Challenger が文脈理解を試す課題と採点基準を作り、Reasoner が skill を使って解き、 Judge が合否を返す。失敗は Reasoner の skill 更新へ、簡単に解けた課題は Challenger の強化へ回ります。

3. 技術や手法のキモは？

1

Challenger が問いを作る

文脈を読めているかを試す課題と採点基準を作ります。解く側だけでなく、問いを作る側にも skill があるのが面白い点です。

2

Reasoner が解く

文脈と現在の Reasoner skill を使って回答します。失敗したら、欠けていた文脈知識を診断して skill を更新します。

3

Judge が合否を返す

採点基準ごとに pass / fail を返します。答えを教える先生ではなく、失敗を検出する圧力として働きます。

4

Cross-Time Replay で選び直す

最後の skill をそのまま採用せず、過去の skill 候補を代表課題で評価し直します。最新が最良とは限らない、というのが大事です。

中核のループ

Challenger 文脈理解を試す課題と採点基準を作る。簡単に解かれたら、次はもっと鋭い問いを作るように更新される。
Reasoner 文脈と現在の skill を使って回答する。失敗したら、足りなかった文脈知識を skill に反映する。
Judge 採点基準ごとに pass / fail を返す。詳しい答えを教える先生ではなく、失敗を検知する圧力として働く。
ポイント: 更新されるのはモデル重みではなく、自然言語の skill set。だから閉じたモデルにも差し込めるし、wiki や AGENTS.md の運用にも近い。

4. どうやって有効だと検証した？

数字は派手ではありません。でも、文脈学習が難しいタスクで一貫して改善している点を見るのがよさそうです。

GPT-4.1 11.1% → 16.5%
GPT-5.1 21.2% → 25.8%
評価カテゴリ 4 種類

絶対性能はまだ低いので、「これで文脈学習は解決」とは読まないほうがいいです。 ただ、skill を外部化して推論時に差し込む方向が、複数モデルで効いているのはかなりおいしい示唆です。

5. 議論はある？

コスト: 各文脈ごとに課題生成、回答、採点、skill 更新を回すので、一回しか使わない文脈には重いです。

Judge 依存: 採点器が文脈理解の深い失敗を拾えるかは別問題です。pass / fail が正しければ強いけど、そこが揺れると危ない。

過剰適応: Cross-Time Replay があるとはいえ、自己対戦は特殊な課題へ寄りやすい。ここは実運用でも同じ匂いがします。

6. 次に読むべき論文は？

作った skill が増えたあと、必要なものをどう検索して使うかを見る次の候補です。Ctx2Skill の後に一番つながります。

AgentSPEX

抽出した skill や手続きを、実行可能な agent workflow の仕様にどう載せるかを見る候補です。

Storage Is Not Memory

wiki や Obsidian に保存するだけでは記憶にならない、という問題意識を深掘りする候補です。

読後Q&A

これは要するに RAG なの？

近いけど違います。RAG は外部文書から根拠を取り出して回答に使うことが中心です。Ctx2Skill は、文脈から規則や手続きを取り出し、次のタスクでも使える skill として保存する方向です。

どうして Challenger にも skill があるの？

Challenger はランダムに問題を出す役ではなく、文脈理解の弱点を突く課題と採点基準を作る役だからです。簡単に解かれた課題は、次にもっと鋭い問いを作るための材料になります。

Cross-Time Replay はなぜ必要？

自己対戦は進めるほど良くなるとは限らないからです。後半の skill は特殊な課題に寄りすぎることがあるので、過去の候補を代表課題で評価し直して選びます。

実務でそのまま使える？

そのまま導入するには重いです。ただ、skill や AGENTS.md を更新するときに、直近の失敗だけでなく代表課題で戻りテストする、という考え方はかなり使えます。
